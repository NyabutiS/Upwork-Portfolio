# Enterprise Data Model – FinTech Claims Processing

## Business Context
A mid-sized FinTech company needed a structured data model to manage customer accounts, policies, claims, and payments efficiently.

## Problem Statement
- Reporting was inconsistent.
- Data was siloed across multiple systems.
- Manual reconciliation caused delays.

## Solution
- Designed **Conceptual → Logical → Physical data model**.
- Built an **ERD** and **Data Dictionary**.
- Generated **SQL DDL scripts** for table creation.

## Deliverables
- ERD Diagram: `ERD.png`
- Data Dictionary: `data_dictionary.pdf`
- SQL Table Creation Scripts: `ddl.sql`

## Tools Used
- ERwin Data Modeler
- SQL Server 2022
- dbdiagram.io

## Business Impact
- Improved reporting consistency.
- Reduced reconciliation errors.
- Foundation for future data warehouse.
